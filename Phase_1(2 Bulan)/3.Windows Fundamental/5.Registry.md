**Registry** adalah database untuk menyimpan config sistem,config apllikasi,setting user,dll.mirip root cuman bedo.intine kyok nggon nyimpen data datamu karo data seng dibutuhno sistem e kanggo ngelayani user,ngono yo?

**Registry** memiliki bentuk seperti pohon(tree) dan isinya:
- Key = folder utama
- Subkey = folder di dalam keynya
- Value = mirip file  yang berisi data-data

**1.Root Key(folder utama)**

![[Screenshot 2026-01-17 101825.png]]
- HKEY_CLASSES_ROOT berisi file asosiasi nama ekstensi file dan informasi pendaftaran kelas(mbuh gamudeng)
- HKEY_CURRENT_USER berisi semua config user yang sedang login
- HKEY_LOCAL_MACHINE berisi configurasi sistem 
- HEKY_USERS berisi config dari ssemua user
- HKEY_CURRENT_CONFIG berisi configurasi Hardware
**2.Value(setiap key pasti punya)**
- REG_SZ = string
- REG_DWORD = angka(32 bit)
- REG_QWORD = angka(64 bit)
- REQ_BINARY = data biner
- REQ_MULTI_SZ = banyak stirng
contoh:
	Name: DisableCMD
	Type: REG_DWORD
	Value: 1

**Cara akses:**
- GUI
	Pencet Windows + R -> ketik  regedit -> wes 
- CLI:
	reg query HKLM\Software

Registry juga terdapat permission juga dan ini biasanya yang di incar oleh attacker.biasanya dipakai di:
- **Persistence**
- **Privilege Escalation**
- **Bypass restriction**
- **Malware config**
