Didalam OS ada yang namanya **program** dan **proses**
**Program** adalan file yang di simpan di disk yang berisi  sebuah instruksi dan belum melakukan apa-apa.contoh **/usr/bin/bash** -> ini namanya program bash
**Proses** adalah instance dari program yang sedang di jalankan.contoh misal buka firefox,akan ada firefox yang akan muncul di layar itu namanya proses.di dalam proses itu ada yang namanya **konteks,resource dan thread**.konteks dan resorce itu di dapat dari sistem berdasarkan program, input dan environtment.sedangkan thread adalah semacam pekerja di dalam proses tersebut.jadi thread akan mengeksekusi instruksi yang sudah ada di dalam konteks tersebut

jika di analogikan dengan OOP,Program adalah Class nya sedangkan Proses adalah Object nya.satu program bisa mempunya banyak proses.


proses dapat dilihat melalui command linux seperti "ps aux".contoh output:

**subahehe   11330  0.0  0.0 233424  5836 pts/1    Ss   22:49   0:00 /bin/bash**
- subahehe = user
- 11330 = PID
- 0.0 = CPU
- 0.0 = RAM 
- 233424=VSZ -> Virtual Memori 
- 5836 = RSS -> Ram yang digunakan.disini 5,8 mb 
- pts/1 = TTY -> berjalan di terminal 
- Ss =STAT -> status **Sleep(menunggu input)** dan session leader,artinya ini proses induk 
- 22.49 = START -> mulai proses
- 0.0 = TIME -> waktu CPU eksekusi 
- /bin/bash = Command untuk path program berada

di dalam satu proses terdapat struktur sebagai berikut:
- **PID**(Process ID) adalah nomor identitas proses yang dimiliki setiap proses.biasanya digunakan sistem untuk kill process,monitor,dll.bisa di cek dengan "ps aux"
- **PPID**(Parent Process ID) child process yang menunjuk ke PID parent
- **Status** adalah process nya lagek lapo.contoh:
	- Running = Berjalan 
	- Sleeping = menunggu input 
	- Stopped = di pause
	- Zombie = Udah mati tapi belum di hapus oleh Process parentnya 
- **Memory Layout** adalah memori dari proses tersebut yang digunakan untuk mengeksekusi program
- Permission(UID/GID) untuk memberi output bahwa process tersebut jalan sebagai user/root.
- CPU Time menujukkan berapa lama menggunakan CPU

**cara kerjanya sederhana,proses akan di buat lalu berjalan menggunakan CPU  & RAM dan menunggu input,biasanya dari disk/network.setelah proses selesai proses akan mati** 

jenis jenis proses
1. **Foregroud Process** adalah proses yang langsung jalan di terminal.contoh nano,cat,ls
2. **Background Process** adalah proses dijalankan di terminal tapi di lepas jadi tidak ada output langsung terminalnya.contoh sleep 
3. **Daemon** adalah process yang jalan secara terus menerus di sisrem.process ini biasanya di akhiri dengan huruf "d".contoh systemd,sshd,bluethoothd,dll.

**SERVICE** adalah pengatur dari process yang dilakukan **daemon**.di dalam file .service biasanya berisi teks untuk mengatur urutan process bekerja,mengatur limit,mengatur unit,dll.
