**Linux** adalah **kernel** open source yang sangat fleksibel untuk dimodifikasi. Kernel adalah sistem inti yang mengatur hardware agar software dapat berjalan. Karena itu, kernel berhubungan langsung dengan hardware. Di dalam kernel terdapat pengatur CPU, memory (RAM), manajemen proses, filesystem, serta framework driver perangkat keras.

Sistem operasi yang menggunakan kernel Linux disebut **distro (distribution)**. Distro adalah sistem operasi yang menggunakan Linux sebagai kernelnya dan dilengkapi dengan userland tools, library, serta konfigurasi. Tiap distro biasanya memiliki racikan berbeda, seperti perbedaan versi kernel, patch, konfigurasi, tingkat stabilitas, keamanan, dan fleksibilitas.

Agar user dapat berkomunikasi dengan kernel, diperlukan suatu **penerjemah** yang menerjemahkan input user menjadi perintah yang dapat diproses sistem. Penerjemah tersebut disebut **shell**. Contoh shell antara lain **bash** dan **zsh**.  
User berinteraksi melalui **terminal**, yang menjalankan shell. Shell kemudian menerjemahkan perintah user menjadi system call yang dapat dipahami oleh kernel.

Cara kerjanya adalah user memberikan input, lalu **shell** memproses dan menerjemahkannya, kemudian perintah tersebut dikirim ke kernel. Kernel akan mengeksekusi program atau perintah yang diminta dan mengembalikan output ke terminal.

**Filesystem** di Linux dimulai dari direktori root (`/`). Direktori ini merupakan akar dari seluruh filesystem dan berisi berbagai folder sistem yang digunakan untuk menjalankan program dan service. Semua data dibagi ke dalam direktori tertentu agar mudah dikelola. Salah satu direktori penting adalah `/home`, yang berfungsi sebagai rumah bagi user. Biasanya user akan berada di direktori ini setelah instalasi dan menyimpan data pribadinya di sana.

Di dalam filesystem Linux terdapat konsep **permission (izin akses)**, yaitu aturan yang menentukan siapa saja yang boleh membaca, menulis, dan mengeksekusi file atau direktori. Permission ditandai dengan `rwx`. Contohnya:

`-rw-r--r--`

Urutannya adalah **owner – group – other**.  
Owner dapat membaca dan menulis file (`rw-`), sedangkan group dan other hanya dapat membaca file tersebut (`r--`).

Di dalam direktori Linux terdapat banyak **program**. Program adalah file yang berisi instruksi, namun belum melakukan apa pun sampai dijalankan. Program yang sedang berjalan disebut **process (proses)**.  
Saat proses berjalan, sistem akan membuat struktur proses yang berisi informasi seperti process ID (PID), user pemilik proses, waktu eksekusi, dan resource yang digunakan.  
Di dalam proses terdapat **thread**, yaitu unit eksekusi yang menjalankan instruksi-instruksi program menggunakan resource yang tersedia.

Di Linux terdapat konsep **privilege**, yang secara umum dibedakan antara **user biasa** dan **root**. Sebelum menjalankan program atau mengakses resource tertentu, sistem akan memastikan privilege dari user atau proses tersebut. Privilege digunakan untuk membatasi tindakan agar sistem tetap aman.  
User biasa memiliki akses terbatas, sedangkan **root (UID 0)** memiliki privilege tinggi dan dapat melewati pengecekan permission file, sehingga dapat melakukan hampir semua tindakan di sistem, kecuali jika dibatasi oleh mekanisme keamanan tambahan seperti SELinux, AppArmor, atau container.
